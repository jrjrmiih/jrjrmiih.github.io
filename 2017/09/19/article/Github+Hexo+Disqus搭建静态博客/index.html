Github+Hexo+Disqus搭建静态博客

发表于 2017-09-19 | 分类于 技术分享 | |

前提条件

  • Git 环境
  • NodeJS 环境

安装 Hexo

$ npm install hexo-cli -g

安装 Git 插件

$ npm install hexo-deployer-git --save

否则部署时会出现错误提示：ERROR Deployer not found: git

配置使用

创建工程

创建工程目录并进入，执行：

$ hexo init

会生成一堆文件和目录，这就是 Hexo 工程的框架。./source/_posts/ 目录中的 *.md 文件就是你博客的文章，是用 MarkDown 语法来书写的。

生成静态页面

创建好的工程，会带一篇叫 hello-world.md 的范例文章，可以用来生成静态页面，用于浏览器展示。命令如下：

$ heox generate

或者简写为：

$ heox g

与 GitHub 配合使用

编辑工程配置文件 _config.yml，将 deploy 字段对应内容配成如下：

deploy:
    type: git
    repo: <你的GitHub 地址，格式为：https://github.com/<用户名>/<用户名>.github.io.git>
    branch: <分支名，如：master>

更换主题

以 NexT 为例，打开其 GitHub 官方地址：https://github.com/iissnan/hexo-theme-next，按照说明安装最新版本：

git clone https://github.com/iissnan/hexo-theme-next themes/next

集成 Disqus 评论系统

随着多说项目的关闭，国内再无出众的评论平台。综合考虑，还得用国外老牌 Disqus 服务，论稳定性那是当仁不让，唯一致命的问题是需要翻墙，但对于 IT 码农来讲，这已是个人电脑的基本必备属性。

首先访问 Disqus，并注册登录。选择 GET STARTED -> I want to install Disqus on my site，然后填写申请表，并在 Website URL 项里填写与其绑定的网址（这里就是 GitHub 的地址，如：https://<用户名>.github.io/）。

然后编辑 Hexo 使用的主题配置文件，位置通常在 ./themes/<主题目录>/_config.yml。这里以 NexT 主题为例（详细内容，见 Hexo 主题官方文档说明），找到 disqus 字段对应内容，配置成如下：

disqus:
  enable: true
  shortname: <disqus 注册名>
  count: true

常见问题

问：执行 Hexo 命令时遇到：

Error: Cannot find module './build/Release/DTraceProviderBindings'

答：卸载 Hexo 后重装一遍，即可解决：

$ npm uninstall hexo-cli -g
$ npm install hexo-cli -g

或加 –no-optional 参数安装：

$ npm install hexo --no-optional
